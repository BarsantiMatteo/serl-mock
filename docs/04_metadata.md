# Dataset Column Reference

Column-level metadata for all SERL Edition 08-style datasets produced by this project.  
For flag value codes see the [Flag codes](#flag-codes) section at the bottom.

## Half-Hourly Smart Meter Data

**File pattern:** `serl_smart_meter_hh_edition08/serl_half_hourly_<year>_<MM>_edition08.csv`  
**Grain:** one row per household per 30-minute UTC period  
**Coverage:** UTC calendar month (00:00:00Z on the 1st to 23:30:00Z on the last day)

| Column | Type | Description |
|---|---|---|
| PUPRN | text | Pseudonymised household identifier (8-character alphanumeric) |
| Read_date_time_UTC | text | Timestamp of the reading in UTC — format `YYYY-MM-DDTHH:MM:SSZ` |
| Read_date_time_local | text | Timestamp in Europe/London local time with timezone label (GMT or BST) |
| Read_date_effective_local | text | Local calendar date the reading belongs to (format `DD/MM/YYYY`); midnight UTC reads roll to the previous local date |
| HH | int (nullable) | Half-hour interval index within the local day (1 = 00:00–00:30, 48 = 23:30–00:00); `NA` if the timestamp is not on a 30-minute boundary |
| Valid_read_time | boolean | `True` if HH is not NA (timestamp is aligned to a 30-minute boundary) |
| Elec_act_imp_hh_Wh | int | Electricity active import for the half-hour period (Wh) |
| Elec_react_imp_hh_varh | int | Electricity reactive import for the half-hour period (varh) |
| Elec_act_exp_hh_Wh | int | Electricity active export for the half-hour period (Wh); non-zero only for households with solar/battery export |
| Elec_react_exp_hh_varh | int | Electricity reactive export for the half-hour period (varh) |
| Gas_hh_m3 | float | Gas consumption for the half-hour period (m³); `0` or small value for households without gas |
| Gas_hh_Wh | int | Gas consumption converted to energy (Wh) using CV = 39.5 MJ/m³ (~10 972 Wh/m³) |
| Elec_act_imp_flag | int | Data quality flag for `Elec_act_imp_hh_Wh` — see [Flag codes](#flag-codes) |
| Elec_act_exp_flag | int | Data quality flag for `Elec_act_exp_hh_Wh` — see [Flag codes](#flag-codes) |
| Elect_react_imp_flag | int | Data quality flag for `Elec_react_imp_hh_varh` — see [Flag codes](#flag-codes) |
| Elect_react_exp_flag | int | Data quality flag for `Elec_react_exp_hh_varh` — see [Flag codes](#flag-codes) |
| Gas_flag | int | Data quality flag for `Gas_hh_m3` — see [Flag codes](#flag-codes) |

## Daily Smart Meter Data

**File pattern:** `serl_smart_meter_daily_edition08/serl_smart_meter_daily_<year>_edition08.csv`  
**Grain:** one row per household per calendar day  
**Note:** daily files are not yet generated by this project (planned)

| Column | Type | Description |
|---|---|---|
| PUPRN | text | Pseudonymised household identifier |
| Read_date_effective_local | text | Local calendar date (format `DD/MM/YYYY`) |
| Read_date_time_UTC | text | UTC midnight timestamp corresponding to the effective date |
| Valid_read_time | boolean | `True` if the daily timestamp is valid |
| Elec_act_imp_flag | int | Data quality flag for daily electricity active import — see [Flag codes](#flag-codes) |
| Valid_hh_sum_or_daily_elec | boolean | `True` if either the daily electricity reading or the sum of half-hourly readings is valid and usable |
| Elec_sum_match | int | `1` if the daily electricity meter read matches the sum of valid HH readings; `0` if there is a discrepancy |
| Gas_flag | int | Data quality flag for daily gas — see [Flag codes](#flag-codes) |
| Valid_hh_sum_or_daily_gas | boolean | `True` if either the daily gas reading or the sum of half-hourly gas readings is valid and usable |
| Gas_sum_match | int | `1` if the daily gas meter read matches the sum of valid HH readings; `0` if there is a discrepancy |
| Elec_act_imp_d_Wh | int | Electricity active import as reported by the daily meter read (Wh) |
| Unit_correct_elec_act_imp_d_Wh | int | Unit-corrected electricity active import daily read (Wh); corrects for known unit-reporting errors in some meter types |
| Elec_act_imp_hh_sum_Wh | float | Sum of valid half-hourly electricity active import readings for the day (Wh) |
| Gas_d_m3 | float | Gas consumption as reported by the daily meter read (m³) |
| Gas_hh_sum_m3 | float | Sum of valid half-hourly gas readings for the day (m³) |
| Gas_d_kWh | float | Gas daily consumption converted to energy (kWh) using the standard calorific value |
| possible_kWh | float | Best available gas energy estimate for the day (kWh); uses the daily read where valid, falls back to the HH sum otherwise |

## SERL Survey Data

**File:** `serl_survey_data_edition08.csv`  
**Grain:** one row per household  
**Source:** Household survey conducted at recruitment; variable definitions in `docs/documentation/serl_survey_data_dictionary_edition07.csv`

| Column | Type | Description |
|---|---|---|
| PUPRN | text | Pseudonymised household identifier |
| Survey_version | text | Survey wave the household completed (Wave1, Wave2, Wave3) |
| Recorded_date | text | Date the survey was recorded (YYYY-MM-DD) |
| Collection_method | text | How the survey was completed (Online, Postal) |
| Language | text | Language in which the survey was completed (English, Welsh, Unknown) |
| A1 | int | Main heating fuel type (coded) |
| A2 | int | Secondary heating type (coded) |
| A3xx | int | Central heating fuel sub-questions (A301–A310; 0/1 binary per fuel type) |
| B1 | int | Type of accommodation (coded; -2 = no answer) |
| B4 | int | Tenure type (coded; -2 = no answer) |
| B5 | int | Number of rooms |
| B9 | int | When accommodation was built (coded decade band) |
| C1 | int | Number of people in the household |
| C1_new | int | Corrected/recoded number of occupants |
| C5 | int | Whether the household has an electric vehicle (-2 to 2) |
| D1 | int | Age of the main respondent (coded band) |
| D2 | int | Employment status (coded) |
| D4 | int | Financial situation self-assessment (coded; -3 to 5) |
| *_err | boolean | Error/edit flag for the corresponding variable (True if a correction was applied) |
| *_sum | int | Derived sum variable (used for multi-part questions) |
| *_text / *_Other | text | Free-text response for open-ended or "other" options |

> Missing value codes: `-1` = don't know, `-2` = no answer / refused, `-9` = not applicable.

## BST Clock-Change Dates

**File:** `bst_dates_to_2030.csv`  
**Grain:** one row per BST transition event  
**Note:** this file is used to identify days with 46 or 50 half-hours when clocks change

| Column | Type | Description |
|---|---|---|
| Read_date_effective_local | text | Local calendar date of the clock change (format `DD/MM/YYYY`) |
| type | text | Direction of the change — `start` (clocks forward, GMT→BST, 46 HH periods) or `end` (clocks back, BST→GMT, 50 HH periods) |
| n_hh | int | Expected number of half-hour periods on that local day (46 on spring change, 50 on autumn change, 48 on all other days) |

## Climate Data

**File pattern:** `serl_climate_data_edition08/serl_climate_data_<year>_<MM>_edition08.csv`  
**Grain:** one row per ERA5 grid cell per UTC hour  
**Note:** climate data files are not currently generated by this project

| Column | Type | Description |
|---|---|---|
| grid_cell | text | ERA5 spatial grid cell identifier (format `<col>_<row>`, zero-padded to 2 digits, e.g. `03_28`) |
| PUPRN | text | Pseudonymised household identifier linked to this grid cell |
| analysis_date | text | Local calendar date (DD/MM/YYYY, e.g. `01/08/2019`) |
| date_time_utc | text | UTC timestamp of the observation (YYYY-MM-DDTHH:MM:SSZ, e.g. `2019-08-01T03:00:00Z`) |
| 2m_temperature_K | float | Air temperature at 2 m above ground (Kelvin; subtract 273.15 for °C) |
| surface_solar_radiation_downwards | float | Accumulated downward solar radiation at the surface (J/m²); divide by 1 800 for average W/m² per half-hour |
| total_precipitation | float | Accumulated total precipitation (metres of water equivalent) |
| 10m_u_component_of_wind | float | Eastward wind component at 10 m (m/s) |
| 10m_v_component_of_wind | float | Northward wind component at 10 m (m/s) |

## Flag codes

All flag columns use the same integer coding:

| Code | Meaning |
|---|---|
| `1` | Valid reading |
| `0` | Missing or null reading |
| `-1` | Meter rollover — value at or above physical meter maximum (16 777 215 Wh for electricity; 16 777.215 m³ for gas) |
| `-2` | Implausibly high value — above the "very high" threshold (24 000 Wh for electricity import; 5 000 Wh for electricity export; 8.0 m³ for gas) |
| `-5` | Invalid read time — timestamp is not aligned to a 30-minute boundary (`Valid_read_time = False`) |
